📄 Overview

This application performs ensemble time-series forecasting of GNSS satellite clock and positional errors at 15-minute intervals using a mixture of neural network architectures and Gaussian Process models.

Ensemble strategy: Simple voting average — each enabled model contributes equally to the final prediction by averaging raw (scaled) outputs before inverse transformation.

📥 Input CSV Format

Your CSV must contain exactly the following columns in any order:

utc_timestringTimestamp in any parseable format (e.g. DD-MM-YYYY HH:mm)
x_error(m)floatX-axis position error in metres
y_error(m)floatY-axis position error in metres
z_error(m)floatZ-axis position error in metres
satclockerror(m)floatSatellite clock error equivalent in metres
Minimum rows required: past_days × 41 (e.g. 7 days → 287 rows minimum)

🧠 Model Reference

Six models are available in the ensemble. Each is loaded from a separate .pth (PyTorch) or .pkl (scikit-learn joblib) file in the model directory.

# GRU — Bidirectional Gated Recurrent Unit hidden_size=98, num_layers=2, dropout=0.3 # biGRU — Bidirectional GRU (larger) hidden_size=107, num_layers=2 # LSTM — Long Short-Term Memory (unidirectional) hidden_size=196, num_layers=2 # biLSTM — Bidirectional LSTM (single layer) hidden_size=115, num_layers=1 # Transformer — Time-Series Transformer Encoder d_model=128, nhead=4, dim_feedforward=409 # GP — Gaussian Process (separate model per feature/timestep) format: dict{"x_error_t0": GPModel, ...}

📤 Output Format

The output CSV mirrors the input column structure with predicted values starting from the next 15-minute interval after the last input timestamp.

utc_time, x_error(m), y_error(m), z_error(m), satclockerror(m) 08-03-2025 00:15, 0.231456, -0.145221, ...

⚠️ Troubleshooting

Common issues and resolutions:

⚠Scaler version mismatch — retrain and re-export the scaler with your current scikit-learn version.
ℹNot enough rows — ensure your CSV has at least past_days × 41 rows before the prediction window.
